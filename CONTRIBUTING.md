# Contributing

Gracias por tu interes en contribuir a este repositorio.

Este proyecto es principalmente un espacio personal de aprendizaje, pero se agradecen mejoras en codigo, documentacion y estructura.

## Como contribuir

1. Abre un issue explicando la mejora o bug.
2. Haz un fork del repositorio.
3. Crea una rama descriptiva.
4. Realiza cambios pequeños y claros.
5. Envia una pull request con contexto suficiente.

## Buenas practicas para cambios

- Respeta la estructura de carpetas existente.
- No subas datos sensibles ni credenciales.
- Evita archivos pesados no necesarios.
- Incluye notas de uso cuando agregues scripts o notebooks.
- Si cambias comportamiento, documentalo en el README del proyecto afectado.

## Estilo recomendado

- Python: codigo legible, nombres claros y funciones pequenas.
- Notebooks: celdas ordenadas y salidas limpias antes de hacer commit.
- Commits: mensajes claros en imperativo.

Ejemplos de mensajes de commit:

- `docs: improve root readme structure`
- `fix: correct preprocessing in datathon script`
- `feat: add baseline model for health dataset`

## Checklist antes de abrir PR

- [ ] El cambio tiene una descripcion clara.
- [ ] El codigo/notebook ejecuta sin errores obvios.
- [ ] No se incluyen secretos, tokens ni datos privados.
- [ ] Se actualiza documentacion cuando aplica.

## Alcance

Debido al caracter mixto del repositorio (academico, hackathons y pruebas), no todos los modulos tienen el mismo nivel de mantenimiento.

Las contribuciones con mas impacto suelen ser:

- Mejoras de documentacion.
- Correcciones de scripts reutilizables.
- Limpieza y organizacion de notebooks.
