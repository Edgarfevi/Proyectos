# Security Policy

## Supported Versions

Este repositorio es de aprendizaje y no tiene versiones LTS formales.

Se intentan corregir vulnerabilidades en la rama principal (`main`) cuando son reportadas.

## Reporting a Vulnerability

Si detectas una vulnerabilidad o exposicion de datos:

1. No la publiques en un issue publico.
2. Reportala por email a: Edgarfevi@outlook.es
3. Incluye pasos de reproduccion y evidencia minima.
4. Si aplica, sugiere impacto y mitigacion.

## Expected Response

Objetivo de respuesta inicial: 7 dias.

Objetivo de evaluacion y accion: 30 dias, dependiendo de la complejidad.

## Scope

Esta politica aplica principalmente a:

- Scripts y aplicaciones dentro de este repositorio.
- Configuraciones que puedan exponer secretos.
- Dependencias con vulnerabilidades conocidas en proyectos activos.

No aplica a entornos locales personales no publicados.

## Good Security Practices

- No subir credenciales, claves o tokens.
- Usar variables de entorno para secretos.
- Revisar archivos antes de commit.
- Mantener dependencias actualizadas cuando sea posible.
